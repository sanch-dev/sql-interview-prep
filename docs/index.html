🧪 Welcome to QueryLab

48 hand-built SQL interview questions in the style asked at top product companies — with a real SQL engine running in your browser.

How it works

  • Pick a question — every one ships with a realistic schema and seed data, loaded into a real SQLite database in your browser.
  • Run (⌘Enter / CtrlEnter) executes your query and shows the result.
  • Submit checks your output against the reference solution — syntax errors, wrong columns, missing rows and value mismatches are all reported.
  • Stuck? Reveal hints one at a time, or open the full solution with an explanation of the pattern being tested.
  • Progress and drafts save automatically in your browser.

Suggested path

Warm up on the easy set (filters, joins, aggregates) → drill mediums until window functions feel automatic → then grind the hard set: gaps & islands, sessionization, retention, interval merging. Those six patterns cover the large majority of senior SQL interview screens.
